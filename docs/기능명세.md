# TXTDrop 기능명세

## 1. 개요

| 항목 | 내용 |
|------|------|
| **앱명** | TXTDrop |
| **버전** | 0.6 |
| **플랫폼** | Windows |
| **형태** | 시스템 트레이 상주 유틸리티 |
| **목적** | 클립보드 내용을 단축키로 파일로 저장 |

---

## 2. 핵심 기능

### 2.1 클립보드 저장

| 클립보드 유형 | 단축키 | 저장 형식 | 파일명 규칙 |
|---------------|--------|-----------|-------------|
| 텍스트 | `Ctrl+Shift+Z` (기본) | `.txt` | `{prefix}_{제목}_{YYYYMMDD_HHMMSS}.txt` |
| 이미지 | `Ctrl+Shift+Z` (기본) | `.png` | `{prefix}_{YYYYMMDD_HHMMSS}.png` |
| 비어 있음 | — | — | 저장 안 함 |

**텍스트 파일명 생성 규칙**

1. **Ollama 사용 가능 시**: AI가 텍스트를 3~5단어로 요약한 제목 사용
2. **Ollama 미사용 시**: 첫 줄에서 Windows 파일명에 사용 가능한 문자만 추출 (최대 40자)
3. **둘 다 실패 시**: `{prefix}_{타임스탬프}.txt`

**저장 폴더**

- 텍스트/이미지 각각 별도 폴더 지정 가능
- 이미지 폴더 미설정 시 텍스트 폴더 사용

---

### 2.2 AI 제목 생성 (Ollama)

| 기능 | 설명 |
|------|------|
| **연동** | Ollama API (`http://localhost:11434`) |
| **역할** | 텍스트 저장 시 파일명용 짧은 제목 생성 |
| **모델** | 설정에서 선택 (llama3.2, phi3, mistral 등) |
| **자동 시작** | 앱 실행 시 Ollama 미실행이면 `ollama serve` 자동 실행 |
| **캐시** | 서버 상태 10초 TTL 캐시 |
| **타임아웃** | 20초 |

**모델 매칭 우선순위**

1. 정확 일치
2. `:latest` 붙인 이름
3. 이름 prefix 일치
4. 첫 번째 사용 가능 모델

---

### 2.3 트레이 메뉴

| 메뉴 | 동작 |
|------|------|
| 환경설정 | 설정 창 열기 |
| 로그 기록 | 로그/저장 기록 창 열기 |
| Ollama 새로고침 | Ollama 상태 확인 및 필요 시 시작 |
| 종료 | 앱 종료 |

- 트레이 아이콘 **좌클릭**: 단축키와 동일 동작 (클립보드 저장)
- **다크 테마** 메뉴 (pystray 네이티브 메뉴 대체)

## 3. 설정 기능

### 3.1 저장 폴더

| 설정 | 기본값 | 설명 |
|------|--------|------|
| 텍스트 저장 폴더 | (첫 실행 시 선택) | 텍스트 파일 저장 경로 |
| 이미지 저장 폴더 | (텍스트와 동일) | 이미지 저장 경로 |

### 3.2 파일명

| 설정 | 기본값 | 설명 |
|------|--------|------|
| 파일명 접두사 | `txtdrop` | 저장 파일명 앞에 붙는 접두사 |

### 3.3 단축키

| 설정 | 기본값 | 제약 |
|------|--------|------|
| 단축키 | `ctrl+shift+z` | Ctrl/Shift/Alt 중 하나 이상 필수 |

### 3.4 AI (Ollama)

| 설정 | 기본값 | 설명 |
|------|--------|------|
| Ollama 모델 | `llama3.2` | 제목 생성에 사용할 모델 |
| 자동 시작 | `true` | 앱 실행 시 Ollama 자동 실행 여부 |

### 3.5 사운드

| 설정 | 기본값 | 설명 |
|------|--------|------|
| 효과음 재생 | `true` | 저장 성공 시 비프음 재생 |

### 3.6 언어

| 설정 | 기본값 | 옵션 |
|------|--------|------|
| 언어 | `ko` | 한국어, English |

- 변경 시 재시작 후 적용

### 3.7 데이터베이스

| 기능 | 설명 |
|------|------|
| DB 백업 | `txtdrop.db`를 지정 폴더에 백업 |
| DB 복원 | 백업 파일로 현재 DB 덮어쓰기 |

## 4. 로그 및 저장 기록

### 4.1 로그 탭

| 항목 | 내용 |
|------|------|
| **레벨** | INFO, WARN, ERROR (색상 구분) |
| **카테고리** | startup, ollama, save, system 등 |
| **보관** | 최근 30일, 5초마다 자동 새로고침 |
| **기능** | 로그 지우기, 새로고침 |

### 4.2 저장 기록 탭

| 항목 | 내용 |
|------|------|
| **컬럼** | 저장 시각, 유형(text/image), 파일명 |
| **동작** | 더블클릭 시 해당 파일 열기 |
| **표시** | 최근 500건 |

---

## 5. 알림 (Toast)

| 유형 | 색상 | 용도 |
|------|------|------|
| 성공 | 녹색 | 저장 완료 |
| 실패 | 빨간색 | 저장 실패 |
| 정보 | 파란색 | AI 제목 생성 중 등 |

- 화면 우하단에 표시
- 5초 후 자동 닫힘
- 클릭 시 로그 창 열기

---

## 6. 시스템 동작

### 6.1 절전 모드 대응

- `WM_POWERBROADCAST` 수신
- 절전 해제 시 단축키 재등록, Ollama 캐시 갱신

### 6.2 첫 실행

- 저장 폴더 미설정 시 폴더 선택 대화상자 표시
- 선택한 경로를 텍스트/이미지 폴더로 설정

---

## 7. 데이터 저장

### 7.1 SQLite DB (`txtdrop.db`)

| 테이블 | 용도 |
|--------|------|
| `config` | 설정 (key-value) |
| `history` | 저장 기록 (시각, 유형, 파일명, 경로) |
| `log` | 로그 (시각, 레벨, 카테고리, 메시지) |

### 7.2 설정 기본값

```
text_save_folder:  ""
image_save_folder: ""
filename_prefix:   "txtdrop"
sound_enabled:     "true"
hotkey:            "ctrl+shift+z"
language:          "ko"
ollama_model:      "llama3.2"
ollama_autostart:  "true"
```
